# 后端代码使用的语言

后端代码的功能是处理来自浏览器的 HTTP 请求，作出响应的回应。

因此后端代码事实上非常自由，可以使用任何语言来编写程序。

你可以将后端代码看作一个函数，输入是请求方法、请求路径等，输出是响应的资源。

## 静态页面

我们提到过，请求路径并不代表实际的文件路径，服务器内部逻辑可以任意实现。

但是有时候，我们希望服务器的内部逻辑就是返回对应路径的文件内容。

这样的页面内容不依赖于任何代码运算结果，也不会改变任何状态，因此称作静态页面。

主流的后端应用例如 Nginx、Apache2 的基本功能就是静态页面的访问。

我们可以将 HTML、CSS、JavaScript 的代码写入文件中，然后作为静态页面放在服务器上。

这样浏览器就可以获取到文件的内容，在本地运行这些代码了。
